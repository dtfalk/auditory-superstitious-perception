  1. CORRECTNESS REVIEW

  The script implements standard RMS normalization with a peak ceiling.
  Here is the analysis of each component:

  a) RMS calculation:
       rms(x, eps=1e-12) = sqrt(mean(x^2) + eps)
     CORRECT. The epsilon prevents division-by-zero in edge cases
     and is negligibly small (1e-12) — it does not distort results.

  b) Gain calculation:
       gain = target_rms / current_rms
     CORRECT. This is a single scalar multiplier applied uniformly
     to every sample:  out[n] = data[n] * gain

  c) Peak ceiling safety:
       max_gain = peak_ceiling / peak_in
       if gain > max_gain: gain = max_gain
     CORRECT. If the desired RMS gain would push peaks above the
     ceiling, the gain is clamped. The file still gets a uniform
     scale — no dynamic compression, no distortion.

  d) Global safe-target computation (main):
     The main() function pre-scans ALL files, computes the maximum
     RMS each file can achieve without exceeding the peak ceiling,
     then picks the MINIMUM of those as the global target. This
     guarantees that EVERY file receives EXACTLY the same gain
     treatment relative to the chosen target — no per-file clipping.
     CORRECT and CONSERVATIVE.

  e) Format preservation:
       sf.write(out_path, out, sr, subtype=info.subtype, format=info.format)
     CORRECT. The output file retains the original sample rate,
     subtype (PCM_16), and format (WAV).

  f) Potential issue — float quantization:
     sf.read() returns float64 samples in [-1, 1]. Multiplying by
     a gain factor and writing back to PCM_16 introduces ±0.5 LSB
     quantization noise. For 16-bit audio this is ~96 dB below full
     scale — completely inaudible and negligible for analysis.

  VERDICT: The script is CORRECT. No bugs found.

----------------------------------------------------------------------
  2. EFFECT ON PEARSON CORRELATION SCORES
----------------------------------------------------------------------

  The Pearson correlation coefficient r is defined as:

       r = Σ((x_i - x̄)(y_i - ȳ)) / sqrt(Σ(x_i - x̄)² · Σ(y_i - ȳ)²)

  Key mathematical property: r is INVARIANT under linear transformations
  of the form  x' = a·x + b  (where a > 0).

  normalize_wavs_rms.py applies:  x' = gain · x  (gain > 0, b = 0)

  Proof that r is unchanged:
    Let x' = g·x.  Then x̄' = g·x̄.
    (x'_i - x̄') = g·(x_i - x̄)
    Numerator:   Σ g(x_i - x̄) · (y_i - ȳ) = g · Σ(x_i - x̄)(y_i - ȳ)
    Denominator: sqrt(g² Σ(x_i - x̄)² · Σ(y_i - ȳ)²)
               = g · sqrt(Σ(x_i - x̄)² · Σ(y_i - ȳ)²)
    r' = [g · num] / [g · den]  =  num / den  =  r

  Therefore: applying the SAME gain to all samples of a signal
  does NOT change its Pearson correlation with any other signal.

  The only caveat is the ±0.5 LSB quantization noise introduced by
  the float→int16 round-trip. For a signal with RMS ~ -23 dBFS
  (≈ 0.071 linear, ≈ 2,326 out of 32,768 levels), quantization
  noise is ~96 dB below signal — the impact on r is on the order
  of 1e-8 or less. This is negligible.

  VERDICT: normalize_wavs_rms.py will NOT meaningfully affect Pearson
  correlation scores. The mathematical invariance of r under uniform
  scaling guarantees this. The quantization noise is negligible.
